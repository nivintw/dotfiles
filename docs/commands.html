Commands

Beyond the third-party tools on The Stack, the repo adds a small set of custom fish functions in home/.config/fish/functions/. The interesting ones are fzf-driven — fuzzy git checkout, live full-text search, a process picker — so the demos below are recorded terminal casts: real runs, scrubbable, no autoplay (recorded in fish; if you selected zsh as your login shell, most of these — everything except the fuzzy-git pickers, fsearch, pset, and the multi-repo helpers — have a zsh port with equivalent behavior).

#Fuzzy git

fzf pickers over git, each with a commit-log preview pane. Type to filter, Enter to act.

⎇

fcor · gcor

Two recency-oriented siblings of fco: fcor lists local branches by most-recent commit; gcor pulls from the reflog, so it surfaces branches you visited even after their remote was deleted.

⌥

gccd

git clone then cd into the new directory in one step — name inferred from the URL, or passed explicitly.

#Search & jump to code

ripgrep into fzf into $EDITOR — pick a match and land on the exact file:line.

⌕

fsearch

The one-shot variant: run ripgrep once for a pattern, fuzzy-filter the results, open the pick. Lighter than fif when you already know the term.

#Processes & the shell

⊗

fkill

Multi-select process picker over ps in fzf: filter, Tab to mark several, Enter to kill. Sends SIGTERM by default (fkill 9 for SIGKILL), and offers to retry under sudo if a kill is denied. (No cast here on purpose — a real recording would show my live process list.)

⌨︎

pset

Set an environment variable from a hidden prompt — the value never touches the command line or shell history. pset GITHUB_TOKEN, type the value, done.

⎘

pubkey

Print an SSH public key and copy it to the clipboard in one move. With no argument it discovers the key from the running agent (1Password, the macOS keychain, or a plain ssh-agent), falling back to ~/.ssh/*.pub; when more than one is found it offers an fzf picker labelled by key comment and SHA256 fingerprint. Pass a path to copy a specific key.

⚑

dnsflush

Flush the macOS DNS cache and bounce mDNSResponder — the two-command incantation you can never remember.

#Many directories at once

Fan a command out across a tree — both handle paths with spaces/newlines safely.

⊞

eachdir · forrepos

The generic engines underneath. eachdir <cmd> runs a command in each immediate subdirectory; forrepos <cmd> runs it at the root of every git repo in the tree (recursively). gs-all/gp-all are just forrepos with a fixed command.

No dry-run, no confirmation
forrepos (and the *-all helpers) run your command in every repo it finds, with no preview. Great for git status; think twice before forrepos git reset --hard.

#Housekeeping

✂︎

git_prune_local

Delete local branches already merged into main — and it's careful about it: detects normal, rebase, and squash merges, and keeps a branch whose remote is gone but whose commits aren't in main yet. --dry-run shows the plan first. This is the one function with its own full bats matrix, because a bug here deletes work.

▤

launch-docs

Serve this very docs site locally (python -m http.server), pick a free port, and open the browser once it's up. How these pages get previewed before they ship.

✚

dotfiles-doctor

Re-derive the intended install end state and report it — Homebrew packages, the login shell, Touch ID for sudo, the firewall, key symlinks, the generated Claude settings, the gitconfig overlay, and that the repo's pre-push hook is still prek's (not silently claimed by another tool) — exiting non-zero if anything needs attention. Reads only and never needs sudo, so it's safe to run any time to answer “is my setup still healthy?” (it runs dotfiles-install --verify, the same checks the install ends with).

#Local AI offload

One command fronting the local Ollama fleet the installer provisions (see The Stack). Unlike the fish functions above it's a plain bash script stowed onto PATH (~/.local/bin/ollm), so it works from any shell — including Claude Code's non-interactive one, its primary caller.

◇

ollm

One-shot generation against a local model picked by role — --role fast|bulk|brainstorm|vision, with the role→model mapping read from scripts/ollama_models.sh so it can never drift from what the installer pulled. Prompt from arguments, context piped via stdin (git diff | ollm "summarize this diff"), --image for the vision model, and only the model's text on stdout so it composes in pipes. It disables hidden thinking by default (reasoning models otherwise return an empty response after burning their token budget) and fails fast with a clear message when the server is down. ollm --list prints the roster with installed state — the same table a SessionStart hook injects into Claude Code so it knows what's available for offload.

The muscle-memory layer, in conf.d/. Abbreviations expand inline as you type (you see the real command before it runs); aliases are interactive-only, so scripts still get the real binaries.
