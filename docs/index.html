Setting up my new computer: clone. run. done.

My actual macOS setup — the same repo I clone onto every Mac I own. home/ mirrors $HOME and is symlinked into place with GNU Stow; everything else — tools, shell, editor, security posture — is reproduced from a single bootstrap script.

And it's built to be forked. The layout and machinery are the reusable part; the package list, the Dock, the keys are mine — yours to swap out. Make it yours →

#Principles

The constraints every change in this repo is held to.

01

Idempotent by default

install.sh is safe to re-run: it converges the machine to the declared state — adopting already-present packages and symlinks instead of clobbering them. The declarative macos.sh and dock.sh steps are the deliberate exception: they re-assert system prefs and rebuild the Dock each run (the Dock step prompts first when interactive).

02

One source of truth

Editing ~/.config/fish/config.fish edits the tracked file directly — Stow makes the dotfile in $HOME and the repo the same file.

03

Secrets never committed

Credentials are 1Password op:// references resolved at install time. gitleaks and a custom SSH-host guard fail the commit if anything concrete slips in.

04

Proportionate testing

Cheap static analysis on every commit, unit tests for the one piece of real logic, and an opt-in VM smoke test that runs the real bootstrap end-to-end. No test heavier than the thing it tests.
