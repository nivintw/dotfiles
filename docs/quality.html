Quality & Testing

Guiding principle: don't build a test suite heavier than the thing it tests. A broken alias costs seconds; a leaked secret or a bricked bootstrap costs real money — spend the budget there.

#The testing pyramid

testing-strategy
many ┌─ Layer 1 · Static analysis (every commit + CI) ─┐ shellcheck, fish -n,
│ lint · secret scan · syntax │ typos, gitleaks, taplo
├─ Layer 2 · Unit + config tests (CI) ───────────┤ bats: fish behavior
│ logic + declarative-config checks │ pytest: config validity
├─ Layer 3 · Integration (CI dry-run + manual VM)┤ stow -n, brew check,
few └─ idempotency · real bootstrap ──────────┘ install.sh on a VM

Run on every commit by prek and re-run identically in CI. Most hooks self-bootstrap a pinned environment.

#Layer 3 & CI

The bootstrap mutates the system, needs sudo/Touch ID, and drives GUI apps — so full automation isn't worth it. CI does the cheap, high-value parts; the honest end-to-end test is a manual run on a throwaway macOS VM (run it twice — the second run is the real idempotency check).

CI — one fast job
A single ubuntu-latest job runs the same prek hooks (not a parallel set of actions), plus bats tests/ and uv run pytest. It pre-installs the few language: system tools it needs (fish, bats, ripgrep); taplo and hawkeye are skipped in CI, and everything else self-bootstraps. Kept green-fast so it runs on every push.

#Licensing

The repo is REUSE 3.3 compliant. Every file that can carry a comment gets an SPDX header, maintained by hawkeye from a single templated source; the handful that can't (strict JSON, the binary iTerm plist, generated lockfiles) are annotated in REUSE.toml instead.
