Getting Started

No prerequisites — install.sh installs Homebrew and uv itself if they're missing. Clone, run, done.

git clone <this-repo> ~/dotfiles
~/dotfiles/install.sh
Idempotent — it converges, it doesn't preserve
Safe to re-run at any point. brew bundle adopts already-installed casks in place, stow refuses to clobber existing real files, and MCP/symlink/shell steps all no-op when already done. The exception is the two declarative system steps: macos.sh re-asserts its preferences and dock.sh rebuilds the Dock from its list every run (the Dock step prompts before doing so interactively) — config-as-code converging the machine to the declared state, not preserving local tweaks.

#The human-only steps

A few things no script can do for you.

›

Claude Code login

The CLI is installed automatically; first run needs a one-time browser sign-in (claude). MCP servers are already registered.

›

GitHub MCP token

Store a PAT in 1Password at op://MCP/github-claude-pat/credential, enable the 1Password CLI integration, then re-run the script — op inject bakes it into ~/.claude.json, never the repo.

›

VS Code & 1Password

Sign in to VS Code to sync, and add the 1Password browser extensions (both apps are casks).

›

AppCleaner SmartDelete

A one-time GUI toggle (installs a privileged helper) — can't be scripted via defaults.

#Per-machine setup

The same public repo runs on every machine; per-machine specifics live in untracked overlay files the bootstrap seeds. Fill in only what a given box needs — see Machine-local overlays for the full pattern.

⊕

Opt-in software bundles

The bootstrap's bundle step shows an fzf multi-select; pick personal and/or homelab. Your choice persists to ~/.config/dotfiles/bundles for idempotent re-runs. To enable one later without re-running the installer:

echo personal >> ~/.config/dotfiles/bundles
brew bundle install --file=~/dotfiles/Brewfile.d/personal.brewfile
⊕

Machine-private software

Work-only or sensitive casks the public repo shouldn't carry go in untracked ~/.config/dotfiles/Brewfile.local — auto-loaded by install.sh, the Homebrew analogue of ~/.gitconfig_local.

⊕

SSH hosts, git identity, fish tweaks

Machine-specific SSH hosts go in ~/.ssh/config.local, a work git identity in ~/.gitconfig_local (e.g. an includeIf "gitdir:~/work/"), and shell tweaks in ~/.config/dotfiles/local.fish. Each is seeded with a commented example.

#Updating

+

New CLI tool / app

Add a brew/cask line to the Brewfile, then brew bundle install --file=~/dotfiles/Brewfile.

+

Dotfile change

Edit the file under home/ (or its symlink in $HOME — same file) and commit.

+

Find untracked installs

brew bundle cleanup --file=~/dotfiles/Brewfile reports prune candidates.

+

New opt-in bundle

Drop a Brewfile.d/<name>.brewfile — hawkeye stamps its SPDX header automatically — then select it on machines that want it.
